**go/README.md specific:**
- Install: `go get cairn.dev/sdk/go` (from go.mod module path)

- Errors: ErrPayloadTooLarge, ErrStoreNotOpen, ErrEmptyTopic, ErrEmptyPayload, ErrImmutabilityViolation (from errors.go)

**ts/README.md specific:**
- Install: `npm install cairn` (placeholder — actual package name TBD)
- Types: Event (id: bigint, topic: string, ts: bigint, payload: Buffer), BatchEvent
- Errors: PayloadTooLargeError, StoreNotOpenError, EmptyTopicError, EmptyPayloadError, ImmutabilityViolationError — each has a `.kind` property
- **BigInt note (IMPORTANT):** All timestamps and event IDs are `bigint`, not `number`. Nanosecond timestamps exceed Number.MAX_SAFE_INTEGER. Use `BigInt(Date.now()) * 1_000_000n` to convert millisecond Date.now() to nanoseconds. Query parameters `start` and `end` are `bigint`.
- Note: Uses better-sqlite3. Synchronous API (no async/await needed).
- Note: Dual ESM/CJS output via tsdown.

**rs/README.md specific:**
- Install: Add `cairn = { path = "..." }` to Cargo.toml (or crates.io name when published)
- Types: Event, BatchEvent, Store, Error enum (from lib.rs)
- **Drop note (IMPORTANT):** Store implements Drop. When a Store goes out of scope, it automatically runs WAL checkpoint and closes the connection. Call `store.close()` explicitly only if you need to handle checkpoint errors. Drop silently discards errors.
- Note: Uses rusqlite with `bundled` feature — builds SQLite from source, no system SQLite needed.
- Note: Store.append and Store.append_batch take `&mut self` (exclusive borrow).
- Note: `Error::kind()` method returns spec-compatible error kind strings.

Keep each README under 150 lines. Technical, concise, no fluff.
