  <action>
Create spec/schema.sql with the exact locked content from CONTEXT.md and RESEARCH.md:

```sql
-- Cairn schema DDL — execute verbatim on every Open()
-- All statements use IF NOT EXISTS for idempotent execution.
-- No PRAGMAs — connection-level settings belong in Open() behavior.

CREATE TABLE IF NOT EXISTS events (
    id      INTEGER PRIMARY KEY,
    topic   TEXT    NOT NULL,
    ts      INTEGER NOT NULL,
    payload BLOB    NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_events_topic_ts ON events (topic, ts);

CREATE TRIGGER IF NOT EXISTS no_update
    BEFORE UPDATE ON events
BEGIN
    SELECT RAISE(ABORT, 'cairn: updates not allowed');
END;

CREATE TRIGGER IF NOT EXISTS no_delete
    BEFORE DELETE ON events
BEGIN
    SELECT RAISE(ABORT, 'cairn: deletes not allowed');
END;
```

Key constraints (all locked):
- INTEGER PRIMARY KEY without AUTOINCREMENT (removes sqlite_sequence write overhead)
- CREATE ... IF NOT EXISTS on all statements (safe to call on every Open)
- Trigger messages are exact strings: 'cairn: updates not allowed' and 'cairn: deletes not allowed'
- No PRAGMAs in this file — they are per-connection and documented in api.md

<task type="auto">
  <name>Task 2: Create API specification document</name>
  <files>spec/api.md</files>
  <action>
Create spec/api.md following the locked document structure from CONTEXT.md. The document MUST follow this exact section order:

1. **Header** — Title, version (use "1.0"), one-paragraph summary of cairn's purpose (append-only immutability)

2. **Concepts** — Formal type definitions:
   - EventID (uint64) — the SQLite rowid returned by INSERT
   - Timestamp (int64) — nanoseconds since Unix epoch
   - Topic (string) — non-empty UTF-8 string identifying an event stream
   - Payload (bytes) — opaque binary data, 1 byte to 1,048,576 bytes (1MB)
   - Event struct { id: EventID, topic: Topic, ts: Timestamp, payload: Payload }

3. **Operations** — One section per operation, each with four parts (signature, preconditions, behavior, errors):

   **Open(path: string) -> Store**
   - Preconditions: path's parent directory must exist
   - Behavior: Open or create SQLite database at path. Execute spec/schema.sql verbatim (CREATE IF NOT EXISTS is idempotent). Set connection PRAGMAs: journal_mode=WAL, busy_timeout=5000, foreign_keys=ON. Enable SQLITE_DBCONFIG_DEFENSIVE via language-binding mechanism (NOT a SQL PRAGMA — this is a C-level sqlite3_db_config call; document per-language: Go modernc via db config API, TypeScript better-sqlite3 via pragma('defensive=ON'), Rust rusqlite via conn.set_db_config(DbConfig::SQLITE_DBCONFIG_DEFENSIVE, true))
   - Errors: path parent directory does not exist -> storage error

   **Close(store: Store)**
   - Preconditions: none (Close is idempotent)
   - Behavior: Execute PRAGMA wal_checkpoint(TRUNCATE) to checkpoint and truncate WAL file. Close database connection. Calling Close on already-closed store is a no-op.
   - Errors: none (idempotent)

   **Append(topic: Topic, payload: Payload) -> EventID**
   - Preconditions: store is open, topic non-empty, payload non-empty and <= 1,048,576 bytes
   - Behavior: Validate preconditions (return error on failure). Get current nanosecond timestamp from system clock. INSERT INTO events (topic, ts, payload) VALUES (?, ?, ?). Return inserted row's INTEGER PRIMARY KEY as EventID.
   - Errors: StoreNotOpen, EmptyTopic, EmptyPayload, PayloadTooLarge

   **AppendBatch(events: []{ topic: Topic, payload: Payload }) -> []EventID**
   - Preconditions: store is open, each event's topic non-empty, each event's payload non-empty and <= 1MB
   - Behavior: If input is empty slice, return empty slice (no error). Validate ALL events before inserting any. Execute all INSERTs in a single transaction (all-or-nothing). Each event gets its own nanosecond timestamp from system clock. Return list of EventIDs in insertion order.
   - Errors: StoreNotOpen, EmptyTopic (first failing), EmptyPayload (first failing), PayloadTooLarge (first failing)

   **Query(topic: Topic, start: Timestamp, end: Timestamp) -> []Event**
   - Preconditions: store is open
   - Behavior: SELECT id, topic, ts, payload FROM events WHERE topic = ? AND ts >= ? AND ts <= ? ORDER BY id ASC. Range is inclusive on both ends [start, end]. Empty result is valid (not an error).
   - Errors: StoreNotOpen

4. **Storage Invariants** — Document each:
   - WAL mode (STOR-01): journal_mode=WAL set at Open, persistent but harmless to re-apply
   - Immutability (STOR-02): no_update and no_delete triggers in schema.sql; triggers use RAISE(ABORT, message)
   - Defensive mode (STOR-03): SQLITE_DBCONFIG_DEFENSIVE prevents disabling triggers via writable_schema; enabled via C-level call, not SQL PRAGMA
   - Nanosecond timestamps (STOR-04): int64 range sufficient; system clock provides timestamps
   - Opaque payloads (STOR-05): cairn never interprets payload content; BLOB column type
   - 1MB payload limit (STOR-06): hard limit at 1,048,576 bytes; validated before INSERT, not by SQLite constraint
   - WAL checkpoint on Close (STOR-07): PRAGMA wal_checkpoint(TRUNCATE) ensures clean state

5. **Error Catalog** — Table format:

   | Code | Operation | Trigger |
   |------|-----------|---------|
   | PayloadTooLarge | Append, AppendBatch | len(payload) > 1,048,576 bytes |
   | StoreNotOpen | Close, Append, AppendBatch, Query | Operation called on closed store |
   | ImmutabilityViolation | (storage-level) | UPDATE or DELETE on events table |
   | EmptyTopic | Append, AppendBatch | topic == "" |
   | EmptyPayload | Append, AppendBatch | len(payload) == 0 |

   Below the table, add per-language idiomatic naming guidance:
   - Go: ErrPayloadTooLarge, ErrStoreNotOpen, etc.
   - TypeScript: PayloadTooLargeError class, StoreNotOpenError class, etc.
   - Rust: Error::PayloadTooLarge variant, Error::StoreNotOpen variant, etc.

Use pseudocode signatures inspired by Protocol Buffers / LSP spec style — clean, language-neutral. Do NOT use any specific programming language syntax.

Resolve open questions from RESEARCH.md:
- 1MB boundary: len(payload) <= 1,048,576 is valid; > 1,048,576 triggers PayloadTooLarge
- Query range: [start, end] both inclusive (WHERE ts >= start AND ts <= end)
- AppendBatch empty: returns empty list, no error
