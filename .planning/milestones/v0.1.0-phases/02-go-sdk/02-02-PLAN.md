  <name>Task 1: Implement Append, AppendBatch, Query</name>
  <files>go/cairn.go</files>
  <behavior>
    - Append("sensor", []byte("hello")) returns (uint64 >= 1, nil)
    - Append("", payload) returns (0, ErrEmptyTopic)
    - Append(topic, []byte{}) returns (0, ErrEmptyPayload)
    - Append(topic, make([]byte, 1048577)) returns (0, ErrPayloadTooLarge)
    - Append on closed store returns (0, ErrStoreNotOpen)
    - AppendBatch([]BatchEvent{{...}, {...}, {...}}) returns ([]uint64 with len 3, nil)
    - AppendBatch([]BatchEvent{}) returns ([]uint64 with len 0, nil) — no transaction
    - AppendBatch with one bad event rejects entire batch
    - Query("sensor", start, end) returns matching events in id ASC order
    - Query("nonexistent", start, end) returns empty slice, nil error
    - Query on closed store returns (nil, ErrStoreNotOpen)
  </behavior>
  <action>
    Replace the stub methods in `go/cairn.go` with full implementations:

    **Append(topic string, payload []byte) (uint64, error):**
    1. `checkOpen()` — return ErrStoreNotOpen if closed
    2. Validate topic non-empty (ErrEmptyTopic), payload non-empty (ErrEmptyPayload), payload <= MaxPayloadSize (ErrPayloadTooLarge). Check in this precedence order.
    3. `ts := time.Now().UnixNano()`
    4. `db.ExecContext(ctx, "INSERT INTO events (topic, ts, payload) VALUES (?, ?, ?)", topic, ts, payload)`
    5. `result.LastInsertId()` — validate id > 0 before casting to uint64
    6. Return `uint64(id), nil`

    **AppendBatch(events []BatchEvent) ([]uint64, error):**
    1. `checkOpen()` — return ErrStoreNotOpen if closed
    2. If `len(events) == 0`, return `[]uint64{}, nil` immediately (no transaction)
    3. Validate ALL events before inserting any. For each event check: topic empty -> payload empty -> payload too large. Return first error found.
    4. `db.BeginTx(ctx, nil)`
    5. Loop: for each event, get ts, INSERT, collect LastInsertId as uint64
    6. `tx.Commit()`
    7. `defer tx.Rollback()` after BeginTx (no-op after Commit)
    8. Return ids slice

    **Query(topic string, start, end int64) ([]Event, error):**
    1. `checkOpen()` — return ErrStoreNotOpen if closed
    2. `db.QueryContext(ctx, "SELECT id, topic, ts, payload FROM events WHERE topic = ? AND ts >= ? AND ts <= ? ORDER BY id ASC", topic, start, end)`
    3. `defer rows.Close()` — CRITICAL for WAL checkpoint
    4. Scan into Event structs, append to slice
    5. Check `rows.Err()` after loop
    6. Return events (empty slice is valid, not error)
  </action>

    func assertErrorKind(t *testing.T, err error, kind string) {
        t.Helper()
        switch kind {
        case "payload_too_large":
            if !errors.Is(err, ErrPayloadTooLarge) { t.Errorf(...) }
        case "empty_topic":
            if !errors.Is(err, ErrEmptyTopic) { t.Errorf(...) }
        case "empty_payload":
            if !errors.Is(err, ErrEmptyPayload) { t.Errorf(...) }
        case "store_not_open":
            if !errors.Is(err, ErrStoreNotOpen) { t.Errorf(...) }

            if !strings.Contains(err.Error(), "updates not allowed") &&
               !strings.Contains(err.Error(), "deletes not allowed") {
                t.Errorf(...)
            }
        }
    }
