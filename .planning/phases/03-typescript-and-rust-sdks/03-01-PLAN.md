Create ts/tests/vectors.test.ts mirroring the Go test harness pattern (go/cairn_test.go). The test file must:

1. **Helper functions:**
   - `readVectorFile(name: string)`: Read from `../../spec/vectors/${name}` relative to test file. Use `fileURLToPath(new URL('../../spec/vectors/' + name, import.meta.url))` for ESM-safe path resolution (not __dirname). Return parsed JSON.
   - `decodePayload(b64: string): Uint8Array`: base64 decode, empty string returns empty Uint8Array. Use Buffer.from(b64, 'base64').
   - `assertErrorKind(error: unknown, expectedKind: string)`: Check that the caught error is an instance of the correct error class by matching its `kind` property.

2. **Open/Close unit tests:**
   - open() creates DB file, WAL mode active
   - open() on missing parent dir throws
   - open() idempotent on existing DB
   - close() idempotent (no error on second call)
   - close() checkpoints WAL (WAL file absent or zero-length after close)

3. **Append vector tests** (from append.json):
   - Iterate test_groups[*].tests[*], for each: open fresh store, handle store_closed flag, handle payload_size_bytes (generate synthetic payload), decode base64 payload, call store.append(), assert valid result (id >= BigInt(event_id_min)) or error (assertErrorKind).

4. **Batch vector tests** (from batch.json):
   - Same pattern. Handle events array, payload_size_bytes per event, store_closed. Call appendBatch(), assert event_id_count or error.

5. **Query vector tests** (from query.json):
   - Setup: insert events with EXACT timestamps from vector file using raw SQL via the store's db handle (NOT via append() which generates its own timestamp). Use `db.prepare('INSERT INTO events (topic, ts, payload) VALUES (?, ?, ?)').run(topic, BigInt(ts), payload)` — timestamps are quoted strings in vectors, parse with BigInt().
   - IMPORTANT: The Store class wraps db privately. Either (a) expose a package-private/test-only accessor, or (b) access the underlying db through a test helper that opens a second better-sqlite3 connection to the same file for raw SQL setup, or (c) add a `_db` getter for testing.    - Query with BigInt start/end parsed from vector strings. Assert returned_count, then compare each event's ts (as bigint) and payload (base64 decoded).

6. **Immutability vector tests** (from immutability.json):
   - Setup: insert events via store.append(). Then execute raw SQL (UPDATE/DELETE) via the underlying db connection. Assert the error message contains the expected string.

Parse all timestamps with BigInt(stringValue) — NEVER Number(). This is critical for precision.
