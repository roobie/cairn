Create the rs/ directory with the complete Rust SDK:

1. **rs/Cargo.toml**: Create with name "cairn", version "0.1.0", edition "2021". Dependencies: rusqlite = { version = "0.34", features = ["bundled"] }. Dev-dependencies: serde_json = "1", base64 = "0.22", tempfile = "3". 2. **rs/src/lib.rs**: Implement the complete SDK:

   - **Error enum**: `#[derive(Debug)]` pub enum Error with variants PayloadTooLarge, StoreNotOpen, ImmutabilityViolation, EmptyTopic, EmptyPayload, Storage(String). Implement `std::fmt::Display` and `std::error::Error` for it. Add a `kind(&self) -> &str` method returning the spec error_kind strings ("payload_too_large", "store_not_open", etc.) for test harness matching.

   - **Constants**: `pub const MAX_PAYLOAD_SIZE: usize = 1_048_576;` and `const SCHEMA_SQL: &str = "..."` (embed spec/schema.sql content verbatim).

   - **Event struct**: `pub struct Event { pub id: u64, pub topic: String, pub ts: i64, pub payload: Vec<u8> }`

   - **BatchEvent struct**: `pub struct BatchEvent { pub topic: String, pub payload: Vec<u8> }`

   - **Store struct**: Contains `conn: Option<rusqlite::Connection>` and `closed: bool`. Using Option<Connection> so Drop can take ownership.

   - **pub fn open(path: &str) -> Result<Store, Error>**: Check parent dir exists (std::path::Path::new(path).parent() + fs::metadata). Open Connection. Apply PRAGMAs via execute_batch (journal_mode=WAL, busy_timeout=5000, foreign_keys=ON). Enable SQLITE_DBCONFIG_DEFENSIVE via conn.set_db_config(DbConfig::SQLITE_DBCONFIG_DEFENSIVE, true). Execute SCHEMA_SQL via execute_batch. Return Store.

   - **Store::close(&mut self) -> Result<(), Error>**: Idempotent. If closed, return Ok. Set closed=true. Run PRAGMA wal_checkpoint(TRUNCATE) (ignore errors). Drop connection by taking from Option.

   - **Store::append(&mut self, topic: &str, payload: &[u8]) -> Result<u64, Error>**: check_open, validate topic/payload/size. Get nanosecond timestamp via SystemTime::now().duration_since(UNIX_EPOCH).unwrap().as_nanos() as i64. Execute INSERT. Return last_insert_rowid() as u64.

   - **Store::append_batch(&mut self, events: &[BatchEvent]) -> Result<Vec<u64>, Error>**: check_open. Empty input returns Ok(vec![]). Validate ALL events first. conn.transaction() -> prepare statement -> loop insert -> collect IDs -> commit. Use `tx.last_insert_rowid() as u64` after each execute.

   - **Store::query(&self, topic: &str, start: i64, end: i64) -> Result<Vec<Event>, Error>**: check_open. Prepare SELECT with WHERE topic=?1 AND ts>=?2 AND ts<=?3 ORDER BY id ASC. query_map to collect Vec<Event>. Return empty vec (not error) for no matches.

   - **impl Drop for Store**: Call self.close() and discard the Result. Must NEVER panic.

   - For query() taking &self (not &mut self): Store needs interior access to conn. Since rusqlite::Connection::prepare takes &self, this works fine — query can take &self.

   - For append/append_batch needing &mut self: transaction() requires &mut Connection. Store methods that mutate take &mut self.

   - Make the Connection field `pub(crate)` or add a `pub(crate) fn conn(&self)` accessor so integration tests can execute raw SQL for immutability vector tests.

Run `cd /home/jani/devel/cairn/rs && rtk cargo build` to verify compilation.
