  <action>
    1. Initialize the Go module:
       - `go/go.mod` with module path matching the repo structure. Use a generic module path like `cairn.dev/sdk/go` or detect from git remote. Require `modernc.org/sqlite` (latest stable).
       - Run `cd go && go mod tidy` to resolve dependencies.

    2. Create `go/errors.go`:
       - Package `cairn`
       - Sentinel errors exactly as spec defines: `ErrPayloadTooLarge`, `ErrStoreNotOpen`, `ErrImmutabilityViolation`, `ErrEmptyTopic`, `ErrEmptyPayload`
       - Error messages must match spec: "cairn: payload too large", "cairn: store not open", "cairn: immutability violation", "cairn: empty topic", "cairn: empty payload"
       - `MaxPayloadSize = 1_048_576` constant

    3. Create `go/cairn.go`:
       - Package `cairn`
       - Import `_ "modernc.org/sqlite"` for driver registration (blank import)
       - `Event` struct: `ID uint64`, `Topic string`, `TS int64`, `Payload []byte`
       - `BatchEvent` struct: `Topic string`, `Payload []byte`
       - `Store` struct: `db *sql.DB`, `mu sync.Mutex`, `closed bool`
       - `schemaSQL` constant: embed the exact DDL from spec/schema.sql as a raw string literal (do NOT use go:embed since the file is outside the module root — copy the DDL verbatim into a const)
       - `Open(path string) (*Store, error)`:
         a. Check parent dir exists with `os.Stat(filepath.Dir(path))` — return storage error if missing
         b. `sql.Open("sqlite", path)`
         c. `db.SetMaxOpenConns(1)` — SQLite single-writer
         d. Execute PRAGMAs via `db.ExecContext`: `PRAGMA journal_mode = WAL; PRAGMA busy_timeout = 5000; PRAGMA foreign_keys = ON; PRAGMA trusted_schema = OFF;`
            Note: `PRAGMA trusted_schema = OFF` is the best available substitute for SQLITE_DBCONFIG_DEFENSIVE via the database/sql interface with modernc.org/sqlite. The immutability triggers provide the actual enforcement. Document this in a code comment.
         e. Execute `schemaSQL` via `db.ExecContext` (idempotent DDL)
         f. Return `&Store{db: db}`, nil
         g. On any error after sql.Open, call `db.Close()` before returning
       - `checkOpen()` method: lock mutex, check closed flag, return ErrStoreNotOpen if closed
       - `Close() error`:
         a. Lock mutex, check if already closed, set closed = true
         b. Execute `PRAGMA wal_checkpoint(TRUNCATE)` — ignore errors (spec: Close is always safe)
         c. Return `db.Close()`
       - Leave Append, AppendBatch, Query as stubs that return `errors.New("not implemented")` — Plan 02 fills these in

    4. Create `go/cairn_test.go`:
       - Package `cairn` (same package for internal access to db field)
       - Write tests per the behavior spec above
       - Use `t.TempDir()` for test isolation
       - For the WAL checkpoint test, do a raw INSERT via `s.db.ExecContext` to put data in WAL, then Close, then check `.db-wal` file size

    5. Run `cd go && go mod tidy && CGO_ENABLED=0 go build ./...` to verify pure-Go build
    6. Run `cd go && go test -v -count=1 ./...` to verify all tests pass
  </action>
