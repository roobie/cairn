## Gap Detail

### STOR-04 / TS-01 / DOC-03 — TypeScript Clock Source Mismatch

**File:** `ts/src/index.ts` lines 160, 187
**Issue:** `process.hrtime.bigint()` returns nanoseconds since an arbitrary system boot reference — not Unix epoch nanoseconds as specified by STOR-04.

**Impact:**
- Timestamps stored by TS SDK are ~10^7 range (boot-relative) vs ~10^18 (Unix epoch)
- Cross-SDK interop breaks: events from TS SDK would sort incorrectly if queried by Go/Rust SDK against the same database
- Vector tests pass because they bypass `append()` for timestamp-sensitive tests

**Fix:** Replace `process.hrtime.bigint()` with `BigInt(Date.now()) * 1_000_000n` (millisecond precision) or `BigInt(Math.trunc(performance.timeOrigin + performance.now())) * 1_000_000n` (sub-millisecond). Update ts/README.md accordingly.
